# CLAUDE.md
## 実行環境の分離
- コンテナはClaude専用の作業場であり、成果物を生成する場所にすぎない
- ユーザーはコンテナ内部で一切作業しない
- 生成した成果物（スクリプト等）の実行環境はmacOSホスト or WSL2 or CI/CDである
- 環境検査・前提確認はコンテナではなく実行環境を基準に行う
## CLAUDE.md
- CLAUDE.md を新規作成する場合:
- 個別具体的な仕様書ではなく、リポジトリ全体に普遍的な構成・規約の索引として保つこと
- 個別具体的な詳細は各ディレクトリに、新たなCLAUDE.mdを作り参照
## 厳守
- memory機能の利用禁止
- 常に日本語で会話
- コード実装はt-wada TDDを利用 / bashはテスト禁止
- ユーザーと問題・課題を認識して合意を取って、会話を行う
## Claudeが利用する SCRATCH MEMO
- `GIT_IGNORED_SCRATCH.md` は global gitignore された memo であり、Claudeが利用
## 禁止
- Markdownで太字の利用を禁止
- 変数やエラーのフォールバック・握りつぶし・暗黙スキップを禁止する
- Bashで ${VAR} 以外の形式を禁止
```
# 禁止
${VAR:-TestVar}
${VAR?Comment}
# 許可
${VAR}
```

## 規則
- 従量課金制で動作している。余計な実装・エラーハンドリングをせず、KISS/YAGNI常に最小限の実装厳守
## 変数命名
- 変数名はtmp等の実装手段でなく、用途を明示
## シェルスクリプト規約
- `func1(){} main(){} main` 型で書く。処理をmainに集約し、ロジックは関数に分離する
## コンテナ規約
- コンテナの内部にファイルを保持する実装は安全ではなく危険である
- 完全に破棄して同一のコンテナが起動できる実装以外を危険とする
- downするたびにnamed volumeを完全に破棄しろ
- 常に `docker down -v` を使え
## セキュリティ規則
- nodeはnpmを禁止. pnpmを使え
- pythonはpipを禁止. uvを使え
- ライブラリに混入したサプライチェーン攻撃を警戒、ライブラリは標準モジュールを最優先、外部ライブラリ回避
## 提案規則
- 技術的制約を理由に、ユーザーへ不便の受容を求めることを禁止する
- 「要件を満たせない」と述べる前に、解決の選択肢を尽くせ。
- 「不可能」は検証済みの事実としてのみ述べる。推論段階では「未確認」と明示する
