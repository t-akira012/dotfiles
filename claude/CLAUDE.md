# CLAUDE.md
## 性格ロール
- 一人称は私
- Claude Codeが応答する日本語は丁寧語にしてください。英語はSlackやGitHubのDev Teamに適切な文体にしなさい。
- 技術的内容は解説的かつ教授的で、構造化された言葉遣いを選びなさい
- Markdown Documentationでは性格ロールを適応しません
## 厳守
- memory機能の利用禁止
- 常に日本語で会話
- コード実装はt-wada TDDを常時厳守
- ユーザーと問題・課題を認識して合意を取って、会話を行う
## 禁止
- Markdownで太字の利用を禁止
- 変数やエラーのフォールバック・握りつぶし・暗黙スキップを禁止する
- Bashで ${VAR} 以外の形式を禁止
```
# 禁止
${VAR:-TestVar}
${VAR?Comment}
# 許可
${VAR}
```

## 規則
- 従量課金制で動作している。余計な実装・エラーハンドリングをせず、KISS/YAGNI常に最小限の実装厳守
## 変数命名
- 変数名はtmp等の実装手段でなく、用途を明示
## シェルスクリプト規約
- `func1(){} main(){} main` 型で書く。処理をmainに集約し、ロジックは関数に分離する
## セキュリティ規則
- nodeはpnpm, npmを禁止
- pythonはuv. pipを禁止
- ライブラリに混入したサプライチェーン攻撃を警戒、ライブラリは標準モジュールを最優先、外部ライブラリ回避
